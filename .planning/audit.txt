1. User query → `detect_intent()` → Intent + Action + Columns
2. Intent → `process_query()` → Pandas operation → DataFrame result
3. DataFrame → `generate_chart_config()` → Chart config + data
4. Result → `generate_explanation()` → LLM response

## 6. AI SYSTEM REBUILD PLAN

### How AI Insights Should Be Generated

1. **Backend-first**: Use Pandas/Scikit-learn for actual computations
2. **LLM for explanation only**: Use LLM to explain results, not compute them
3. **Deterministic fallbacks**: When LLM fails, return computed data with fallback message

### What Data AI Should Use

- **Use**: Profile data (column types, stats), computed aggregations, chart data
- **Don't Use**: Raw user queries for decision-making, unvalidated filters

### LLM Usage Boundaries

| Use Case | LLM Appropriate? | Implementation |
|----------|-----------------|----------------|
| Explain query results | Yes | `generate_explanation()` |
| Generate follow-up questions | Yes | Return in JSON from LLM |
| Auto-insights summary | Yes | `auto_insights_with_kimi()` |
| Compute aggregations | NO | Use Pandas in query_engine |
| Detect intent | NO | Use regex/rules in query_engine |
| Chart type selection | NO | Use rules in query_engine |
